Precision ET is an intelligent irrigation scheduling feature for Toro's consumer IoT platform, Oasis. It translates the scientific FAO-56 Penman-Monteith evapotranspiration equation into a consumer-friendly mobile wizard, automatically generating optimized irrigation schedules based on hyper-local weather and property characteristics.

We designed and built this feature across iOS (SwiftUI), Android (Jetpack Compose), and a .NET admin console. The entire pipeline—from competitive UX research to Figma screen design and native implementation—was orchestrated through Vectoro Bridge V2, a custom AI-to-Figma engine, ensuring pixel-perfect fidelity down to 0.5px strokes.

The Problem

Bridging the Gap Between Science and Homeowners

Toro's Oasis platform connects residential irrigation controllers to a mobile app. The existing app handled basic manual scheduling—start times, durations, days of the week. But modern competitors (Rachio, Hunter Hydrawise) were shipping weather-responsive smart scheduling that adjusts watering based on real-time evapotranspiration data.

Toro already had this technology in their commercial platform, serving golf courses and large properties. The challenge: bring that same scientific rigor to a consumer-friendly mobile experience without the complexity that enterprise users tolerate.

Research

Competitive Intelligence at Scale

We conducted a comprehensive UX teardown of 8 significant smart irrigation platforms. The finding was clear: every competitor collects the same 5-6 zone attributes through nearly identical form flows, but none of them do a good job of explaining why each input matters, showing the impact of selections in real-time, or making configuration feel rewarding.

We established three design mandates:

  • Educate: Explain why each category matters for irrigation in plain language.
  • Visualize impact: Show real-time feedback as users make selections.
  • Celebrate completion: Make profile building feel like an achievement, not a chore.

Research moved from competitor teardown to design principles: clearer education, more visible system feedback, and a setup flow that feels less like configuration debt.

The Process

The Design Engine: Vectoro Bridge V2

This wasn't a standard design handoff. The entire pipeline was orchestrated through Vectoro Bridge V2, a custom-built Model Context Protocol (MCP) server that gives AI assistants direct, programmatic access to Figma. It let our AI agents read, write, create, and manipulate every element in the Figma file in real time.

With 71 tools spanning token application to auto-layout flexbox control, the AI designed production screens utilizing exact components from Toro's enterprise Design System. No hardcoded colors. No random spacing. The Figma token was law.

The flow was designed as a living system, not a static mock. AI-assisted Figma generation, token control, and native QA all fed the same source of truth.

Solution Walkthrough

The 7-Step Precision ET Wizard

The wizard guides users through zone characterization with educational callouts and direct feedback. Each step maps exactly to a scientific variable in the FAO-56 equation:

The wizard turns irrigation science into a step-by-step consumer flow. Each decision teaches just enough, keeps momentum high, and progressively builds a schedule the user can trust.

1

Start with the irrigation hardware

Users first identify the zone hardware, giving the system the base precipitation behavior it needs for downstream calculations.

2

Capture the plant type

Vegetation selection maps to crop coefficient values, but the UI translates that decision into familiar plant language instead of agronomy jargon.

3

Choose the soil profile

Soil type drives infiltration and retention, so this step helps the schedule behave realistically instead of treating every yard the same.

4

Adjust for sun and slope

Exposure and terrain refine the runtime model, letting the wizard explain why certain yards need different watering strategies.

5

Generate an understandable schedule

The system converts scientific values into gallons, runtime, and cadence without forcing homeowners to interpret the equation themselves.

6

Apply the setup across every zone

Once the profile is complete, users can apply those settings across as many zones as they have, turning one guided setup into a scalable scheduling action for the whole property.

Implementation

From Figma to Pixel-Perfect Native

The pixel-perfect process was ruthless. For every screen, we extracted the Figma node tree via REST API and cross-referenced it against SwiftUI/Jetpack Compose code to compare declared paddings, font sizes, and colors.

We executed 202 checkpoint iterations. If the Figma spec said 12px, the code said `.padding(.top, 12)`. Not 11. Not 13. By using AI to extract discrepancy reports, we achieved exact iOS and Android parity across 43 unique UI components.
